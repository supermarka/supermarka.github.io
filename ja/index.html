画像EXIF・フォーマット検査ツール

写真や画像ファイルを下にドロップすると、実フォーマット(拡張子ではなくファイル署名で判定)、ピクセルサイズ、ファイルサイズ、色の種類、さらにカメラのメーカー・機種、露出、ISO、GPS などの EXIF メタ情報を読み取れます。すべてブラウザの FileReader API でローカル解析し、画像をアップロードすることはありません。

プライバシー:画像はブラウザ内だけで読み取り・解析され、どのサーバーにもアップロードされません。

対応フォーマットと「署名による判定」の仕組み

ビューアは拡張子からフォーマットを推測しがちですが、拡張子は自由に変更できます。より確実なのは、ファイル先頭の数バイトの署名(マジックナンバー)を読むことです。本ツールが判定する主な画像フォーマットと署名は次のとおりです:

EXIF メタ情報とは何か、どう読まれるか

EXIF(Exchangeable Image File Format)は、カメラやスマホが写真に書き込むメタ情報で、JPEG・TIFF・一部の HEIC によく含まれます。多くは JPEG の APP1 セグメント内に、TIFF の IFD ディレクトリ構造で、カメラのメーカー・機種・撮影日時・露出・絞り・ISO・焦点距離を保持し、ときに GPS の緯度経度も含みます。スクリーンショットや Web 圧縮画像、多くの PNG は EXIF が削除されています。

2026 年の画像フォーマットとメタ情報の推計

推計・一次実測ではありません
  • 2026 年時点の推計:JPEG は依然として Web とカメラ出力で最も一般的なフォーマットで、Web 画像でも上位を占めます。
  • 2026 年時点の推計:2017 年以降 iPhone は既定で HEIC 撮影となり、近年の機種は主に HEIC を出力するため、Windows では変換が必要なことが多いです。
  • 2026 年時点の推計:Web 画像での WebP・AVIF の採用は増え続けていますが、カメラ側ではまだ少数です。
  • 2026 年時点の推計:スマホ 1 枚の写真は 2〜5 MB・1200 万〜5000 万画素が一般的で、SNS にアップすると再圧縮され EXIF が削除されることが多いです。

よくある質問

画像はサーバーにアップロードされますか?

いいえ。ブラウザの FileReader API でローカルに読み取り・解析するため、画像は端末から出ず、どのサーバーにもアップロードされません。

なぜ私の画像に EXIF がないのですか?

スクリーンショット、Web からダウンロードして再圧縮された画像、多くの PNG には通常 EXIF がありません。SNS もアップロード時にプライバシー保護のため EXIF を削除します。

拡張子と「実フォーマット」が違うのはなぜ?

拡張子はファイル名の一部にすぎず、自由に変えられます。本ツールはファイル先頭の署名バイトを読むため、.png を .jpg に変えても正しく判定します。

撮影場所は分かりますか?

写真が GPS 情報を保持していれば(スマホの未編集オリジナルに多い)、座標を表示します。SNS で圧縮された画像はたいてい GPS がありません。

HEIC / RAW に対応していますか?

署名で HEIC/HEIF などを判定しファイル情報を表示します。ブラウザの HEIC デコード対応は限定的で、サムネイルは表示されないことがあります。Windows で快適に見るには 2345看图王 が使えます。

データは正確ですか?

フォーマット署名・サイズ・EXIF 項目はファイルから直接読むため信頼できます。「2026 推計」の節は傾向推計で、一次実測ではないと明記しています。
