Image EXIF & Format Inspector

Mag-drop ng larawan o image file sa ibaba para basahin ang totoong format nito (natukoy ng file signature, hindi lang ang extension), pixel dimension, laki ng file, uri ng kulay, at EXIF metadata gaya ng camera at model, exposure, ISO at GPS. Lokal na na-parse ang lahat sa iyong browser gamit ang FileReader API — walang na-upload na larawan.

Mag-drop ng larawan dito, o i-click upang pumili

Privacy: ang mga larawan ay binabasa at na-parse lamang sa loob ng iyong browser at hindi kailanman ina-upload sa anumang server.

Mga sinusuportahang format at kung paano gumagana ang signature detection

Madalas na hinuhulaan ng mga tumitingin ng larawan ang format mula sa extension ng file, ngunit maaaring palitan ng pangalan ang isang extension sa kahit ano. Ang isang mas maaasahang diskarte ay basahin ang unang ilang signature byte (ang magic number). Inililista ng talahanayan sa ibaba ang mga pangunahing format ng imahe na nakikita ng tool na ito at ang kanilang mga lagda:

Ano ang EXIF metadata at kung paano ito binabasa

Ang EXIF (Exchangeable Image File Format) ay isang block ng metadata na isinusulat ng mga camera at telepono sa isang larawan, karaniwang nasa JPEG, TIFF at ilang HEIC file. Karaniwan itong nasa JPEG APP1 segment, na nakaimbak bilang mga direktoryo ng TIFF IFD na may hawak na camera make, model, capture time, exposure, aperture, ISO at focal length — at minsan GPS latitude at longitude. Ang mga screenshot, web-compress na larawan at karamihan sa mga PNG ay karaniwang natanggal ang kanilang EXIF.

2026 na pagtatantya para sa mga format ng larawan at metadata

pagtatantya · hindi unang-kamay na pagsukat
  • Pagtantiya noong 2026: Ang JPEG ay nananatiling pinakakaraniwang lalagyan para sa web at mga straight-out-of-camera na mga kuha, na nangunguna pa rin sa mga larawan sa web.
  • Tantyahin noong 2026: mula noong 2017 ang iPhone ay nag-shoot sa HEIC bilang default, at ang mga kamakailang modelo ay naglalabas ng karamihan sa HEIC, na kadalasang nangangailangan ng transcoding sa Windows.
  • Pagtantiya noong 2026: Ang paggamit ng WebP at AVIF sa mga larawan sa web ay patuloy na tumataas, habang bihira pa ring gamitin ng mga camera ang mga ito.
  • Tantyahin noong 2026: ang isang larawan ng telepono ay karaniwang 2–5 MB sa 12–50 megapixel; ang mga social platform ay karaniwang muling i-compress ang mga pag-upload at hinuhubad ang EXIF.

Mga madalas itanong

Na-upload ba ang aking mga larawan sa isang server?

Hindi. Ginagamit ng tool ang browser na FileReader API upang magbasa at mag-parse ng mga imahe nang lokal; hindi kailanman umaalis ang larawan sa iyong device at hindi ina-upload sa anumang server.

Bakit walang EXIF ang aking imahe?

Mga screenshot, mga larawang na-download at muling na-compress mula sa web, at karamihan sa mga PNG ay karaniwang walang EXIF; Tinatanggal din ng mga social platform ang EXIF ​​​​sa pag-upload upang maprotektahan ang privacy.

Bakit iba ang extension sa totoong format?

Ang isang extension ay bahagi lamang ng pangalan ng file at maaaring malayang palitan. Binabasa ng tool na ito ang mga signature byte sa simula ng file, kaya kahit isang .png na pinalitan ng pangalan sa .jpg ay natukoy nang tama.

Maaari ko bang makita kung saan kinuha ang isang larawan?

Kung pinanatili ng larawan ang GPS data nito (karaniwan para sa hindi na-edit na mga orihinal na telepono), ipinapakita ng tool ang mga coordinate. Karamihan sa mga larawang na-compress ng mga social platform ay hindi na nagdadala ng GPS.

Sinusuportahan ba nito ang HEIC / RAW?

Nakikita ng tool ang HEIC/HEIF at mga katulad na format sa pamamagitan ng lagda at nagpapakita ng impormasyon ng file; Ang suporta sa browser para sa pag-decode ng HEIC ay limitado, kaya maaaring hindi mag-render ang isang thumbnail. Para sa maayos na pagtingin sa Windows maaari mong gamitin ang 2345 Picture Viewer.

Ang data ba ay tumpak?

Ang mga format ng lagda, dimensyon at EXIF ​​na field ay direktang binabasa mula sa file at maaasahan; ang seksyon ng mga pagtatantya sa 2026 ay isang pagtatantya ng trend at malinaw na nilagyan ng label bilang hindi unang-kamay na pagsukat.
